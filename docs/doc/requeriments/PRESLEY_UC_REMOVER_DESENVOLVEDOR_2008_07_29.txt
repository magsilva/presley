Use-Case: Remover Desenvolvedor
1 Descrição Breve
Descreve como o desenvolvedor/gerente remove dados de outros desenvolvedores do sistema.
2 Ator
2.1 Desenvolvedor/Gerente
3 Pré Condições
Não existir associação entre um desenvolvedor e uma tarefa não concluída.
4 Fluxo Básico de Eventos
1. O desenvolvedor/gerente seleciona a listar desenvolvedores.
2. O sistema exibe a lista com os desenvolvedores cadastrados no sistema
3. O desenvolvedor/gerente seleciona o desenvolvedor a ser excluído.
4. O sistema verifica se existe associação entre uma tarefa não concluída e o desenvolvedor a ser removido.
5. O sistema remove os dados do desenvolvedor.
6. O sistema avisa ao desenvolvedor/gerente que os dados foram removidos com sucesso.
7. Caso de uso finalizado com sucesso.
5 Fluxo Alternativo
5.1 Existe associação entre um desenvolvedor.
Se no passo 4 do fluxo básico existir associação entre uma tarefa não concluída e o desenvolvedor a ser excluído, 
1. O sistema avisará ao desenvolvedor/gerente que o desenvolvedor não pode ser excluído.
2. Retorna ao passo 3.
6 Pós Condições 
6.1 Desenvolvedor removido com sucesso
O desenvolvedor foi removido com sucesso.
